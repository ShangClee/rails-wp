# AdminWP - Admin Console Documentation

**Last Updated:** Thu Mar 05 2026

## Overview

AdminWP is a modern admin console for managing the Rails-WP monorepo. It provides a tabbed interface with two main sections: **CMS** (content management) and **System** (backend administration).

## Access

- **URL:** `http://localhost:8080/adminWP/index.htm`
- **Hash Routing:** Uses hash-based navigation (e.g., `#cms/posts`, `#system/users`)

## Navigation Structure

### CMS Tab (FrontWP Focused)

| Section | Description |
|---------|-------------|
| [Posts](cms-posts.md) | Create, edit, and manage blog posts |
| [Pages](cms-pages.md) | Manage static pages |
| [Media](cms-media.md) | Media library for images and files |
| [Menus](cms-menus.md) | Navigation menus and structure |

### System Tab (BackWP Focused)

| Section | Description |
|---------|-------------|
| [Users](system-users.md) | User management (add, edit, delete) |
| [Roles](system-roles.md) | User roles and capabilities |
| [Tokens](system-tokens.md) | API token management |
| [Health](system-health.md) | System health and status |
| [Settings](system-settings.md) | Global site settings |

## Features

- **Tabbed Interface** - Clean, organized navigation
- **Hash-based Routing** - Browser history support
- **Responsive Design** - Works on desktop and mobile
- **Toast Notifications** - Feedback for user actions
- **Skeleton Loaders** - Loading states for data fetching

## API Integration

The admin panel communicates with the backend API at `http://localhost:8888/api/v2/`.

### Authentication

- JWT token stored in `localStorage` under key `jwt_token`
- Token passed in `Authorization` header

### Available Endpoints

- `GET/POST /api/v2/posts` - Posts CRUD
- `GET/POST /api/v2/users` - Users CRUD
- `GET /api/v2/setup` - Setup status

## Development

To modify the admin panel:

1. Edit `index.htm` for the main layout
2. Edit `admin-shell.js` for the shell logic
3. Edit individual modules in `shared/js/modules/`

## Legacy Pages

All legacy `.html` pages in `cms/` and `system/` directories now redirect to the main admin shell for a unified experience.
